# 爆款标题拆解

## 目标

把单条标题拆成“为什么会爆”与“怎么迁移到用户自己的业务”两部分。这个 Skill 面向任何用户，不默认他们是谁；若没有业务定位，先补定位，再做改写。

## 总流程

```text
检查是否已有业务定位卡
        ↓
没有 → 先引导填写业务定位
        ↓
生成业务定位卡
        ↓
请用户提供待拆解标题
        ↓
输出拆解表格 + 适合该用户业务的改写标题
```

## 第一步：先检查是否已有业务定位

先判断当前上下文里是否已经有足够的定位信息，至少包括：
- 用户是谁
- 卖什么
- 想吸引谁
- 用户最想解决什么
- 用户最深的焦虑
- 不想吸引谁
- 内容边界

若这些信息已经足够，直接进入“请用户提供待拆解标题”。
若不足，不要急着拆标题，先进入定位生成与校准。

## 第二步：没有定位时，先做定位生成与校准

读取 `references/业务定位采集.md`。
不要一上来就让用户填完整问卷。先请用户用自然语言说一说自己的方向；基于这句话，先生成一版“初始定位草案”，再用少量关键追问把它校准到可用。

完成后，先输出一份《业务定位卡》，再告诉用户：
“定位已完成。现在把你想拆解的标题发给我。”

## 第三步：收到标题后再拆解

读取 `references/拆解标准.md`。
若用户附了链接或截图，优先结合全文或截图判断；若信息不足，明确写“需看全文确认”，不要把猜测写成事实。

默认用两列表格输出：`字段 | 拆解`。
按以下 8 项依次给出：
1. 选题
2. 人群
3. 痛点
4. 反常识观点
5. 情绪
6. 传播点
7. 钩子公式
8. 改写

其中“改写”必须同时给出：
- 贴图版
- 长文版

## 改写原则

- 先判断“这条在卖什么心理”，再判断“它用了什么句式”。
- 改写必须基于用户自己的业务定位，而不是默认迁移到某个固定赛道。
- 改写时，不只迁移句式骨架，还要尽量迁移原标题真正的传播动能，例如高结果诱惑、强身份代入、清单感、站队感、稀缺信息感。
- 如果句式和传播力不能同时保留，优先保住“为什么它会被点开、收藏、转发”的那股力。
- 不要为了追求流量，把标题改写成会吸引错误客户的方向。
- 风格直接、准、不水，像参谋，不像讲义。


## 表格入库规则（可选）

默认只在当前对话里输出拆解表格，不要自动写入任何飞书 / Lark 表格、Excel、数据库或第三方账号。

只有当用户明确提出“存到飞书表格 / 追加到表格 / 导入我的表格”时，才进入入库流程：

1. 先确认当前环境是否具备飞书 / Lark 表格写入能力，例如已安装并配置 `lark-sheets`、飞书表格 Skill、飞书连接器，或用户提供了可调用的表格工具。
2. 如果没有这些能力，明确告诉用户：需要先安装并配置飞书表格相关 Skill / 连接器，才能写入飞书表格。不要假装已经保存。
3. 如果具备写入能力，必须让用户提供自己的飞书表格 URL / token，并确认目标 sheet 与字段结构。
4. 只写入用户明确提供或当前账号有权限访问的目标表格；不要使用开发者、作者、示例文档或历史会话里的表格地址。
5. 写入前先向用户确认本次要追加的行内容；写入后返回目标表格链接或写入位置。

推荐表头：`原标题｜选题｜人群｜痛点｜反常识观点｜情绪｜传播点｜钩子公式｜改写-贴图版｜改写-长文版`。

## 快速判断顺序

1. 用户的定位是否足够支撑改写？
2. 这条标题真正讲什么？
3. 谁会立刻觉得“这说的是我”？
4. 她为什么会点开？
5. 它反了哪条默认常识？
6. 它点燃了什么情绪？
7. 为什么有人会转发 / 收藏？
8. 哪一句是可复用骨架？
9. 怎么把这套力学搬到用户自己的业务里？
